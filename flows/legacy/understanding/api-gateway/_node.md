# Understanding: API Gateway

> FastAPI REST interface for Tor proxy management

## Phase: EXPLORING

## Hypothesis

The API Gateway provides HTTP endpoints for creating, listing, and destroying Tor SOCKS proxies. It interfaces directly with the Docker daemon to spawn/kill containers and reads exit node data from SQLite.

- `api/main.py` - Core FastAPI application (296 lines)

## Validated Understanding

### Core Functionality

The API Gateway (`api/main.py`) is a **FastAPI application** that serves as the control plane for Tor proxy lifecycle management.

#### Configuration Constants
```python
DB_PATH = "/app/db/tor_nodes.db"        # Exit node database
TOR_IMAGE_NAME = "tor-proxy-image:latest"
API_HOST_PORT = 8000
CONFIGS_INTERNAL_PATH = "/etc/tor_configs"
CONFIGS_VOLUME_NAME = "socks-proxy_tor_configs_data"
```

#### REST Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/proxies` | List all running proxy containers with metadata |
| POST | `/proxies` | Create new proxy with geo_category (US/NON_US) |
| DELETE | `/proxies/{port}` | Terminate proxy by host port |
| GET | `/health` | Health check, refreshes metrics |
| GET | `/metrics` | Prometheus metrics (via instrumentator) |

#### Data Models

```python
class CreateProxyRequest:
    geo_category: str       # "US" or "NON_US"
    fixed_ip_mode: str      # "flexible" (fingerprint) or "strict" (IP)

class ProxyInfo:
    port: int               # Host-mapped SOCKS port
    geo_category: str
    exit_ip: str            # Selected exit node IP
    fingerprint: str        # Tor relay fingerprint
    container_id: str       # Docker container ID
    status: str             # Container status
```

#### Key Behaviors

1. **Proxy Creation Flow**:
   - Query SQLite for random exit node matching geo_category
   - Generate torrc configuration with ExitNodes directive
   - Write torrc to shared volume (`/etc/tor_configs/`)
   - Launch Docker container with mounted config
   - Return dynamically assigned host port

2. **Container Management**:
   - Uses `docker` Python SDK for container operations
   - Containers named: `tor-proxy-{fingerprint[:12]}-{timestamp}`
   - Metadata stored in container labels (`com.example.proxy_info`)
   - Containers auto-remove on stop (`remove=True`)

3. **Observability Integration**:
   - `prometheus-fastapi-instrumentator` for HTTP metrics
   - Custom gauges: `active_proxies_gauge`, `available_nodes_gauge`
   - Metrics refreshed on health check and proxy operations

#### Security Considerations

- **Docker Socket Access**: API has full Docker daemon access via socket mount
- **No Authentication**: Endpoints are unauthenticated (internal service assumption)
- **File Write**: API writes torrc files to shared volume
